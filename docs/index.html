What gets checked

Existence

Flags entries whose titles do not match credible records in the enabled sources.

Metadata

Compares authors, year, venue, DOI, pages, volume, and publisher fields.

Export edits

Shows a four-view editor with Original, Found, Edit, and Diff columns.

Notes

Privacy

Your full file is parsed in the browser. Verification sends lookup queries only to sources you enable.

Missing records

Workshop papers, theses, private reports, or very recent publications may need manual checking.

Rate limits

Public APIs are queried at a controlled pace. You can pause, resume, and review completed entries while verification continues.

Guide
Workflow guide
  1. Settings: choose sources and optional API keys.
  2. Parse first: upload or paste, then inspect Parsed/Error entries.
  3. Start: verify parsed entries; Pause waits for the active request.
  4. Log: expand it to inspect requests, retries, and decisions.
  5. Filter: click status badges to focus on one result group.
  6. Candidates: click another match to recalculate suggestions.
  7. Edit: click Original/Found values, type in Edit, or clear fields.
  8. Export: download anytime; unresolved entries are warned, not blocked.
Search Sources (Tiered)
Layered search: Tier 1 (published records) → Tier 2 (conference proceedings) → Tier 3 (repositories and preprints). Zenodo is queried only when the entry already mentions Zenodo.
